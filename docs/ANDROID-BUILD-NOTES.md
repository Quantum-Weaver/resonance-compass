# Android Build Notes — Resonance Compass v2

`src-tauri/gen/` is **gitignored** and fully regenerated by `npm run tauri android init`.
Every customization below lives inside `gen/` and is silently lost on regeneration —
re-apply this checklist after any `android init`.

## After `tauri android init` — required steps

### 1. Regenerate app icons (cello sigil)

`android init` stamps the default Tauri icons into
`gen/android/app/src/main/res/mipmap-*/`. Restore the sigil:

```
npm run tauri icon src-tauri/icons/source.png
```

If the CLI writes to `src-tauri/icons/android/` instead of `gen/`, copy that tree into
`gen/android/app/src/main/res/` (including `mipmap-anydpi-v26/` and
`values/ic_launcher_background.xml`).

### 2. Re-add storage permissions to AndroidManifest.xml

`gen/android/app/src/main/AndroidManifest.xml` — after the INTERNET permission:

```xml
<!-- Android 13+ audio access (API 33+) -->
<uses-permission android:name="android.permission.READ_MEDIA_AUDIO" />
<!-- Android 12 and below storage access -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" android:maxSdkVersion="32" />
```

Without these, the Settings permission toggle never appears and library scanning
finds nothing (see below).

### 3. Re-apply MainActivity edge-to-edge (v1 parity)

`gen/android/app/src/main/java/com/audhd/resonance_compass/MainActivity.kt`:

```kotlin
import androidx.activity.enableEdgeToEdge

class MainActivity : TauriActivity() {
  override fun onCreate(savedInstanceState: Bundle?) {
    enableEdgeToEdge()
    super.onCreate(savedInstanceState)
  }
}
```

## How library scanning works on Android

`tauri-plugin-dialog` **cannot** open a folder picker on mobile — its mobile branch
returns `FolderPickerNotImplemented` (verified in plugin source, v2.7.1). So on
Android the app skips the dialog and scans the standard public locations directly:
`/storage/emulated/0/Music` and `/storage/emulated/0/Download`
(`ANDROID_MUSIC_DIRS` in `src/lib/stores/library.svelte.ts`).

Plain-path reads of public media files work through scoped storage's FUSE layer
once **Music and audio** (API 33+) / **Files** (≤32) permission is granted. The app
declares the permissions but does not request them at runtime (no
runtime-permission bridge exists in our plugin set — same as v1's design); the
vessel grants access manually:

> Settings → Apps → Resonance Compass → Permissions → Music and audio → Allow

The scan surfaces two failure signals in the UI:
- `PERMISSION_DENIED: …` — directory read refused (older Androids) → guidance card.
- Zero tracks found — scoped storage hides unpermitted media instead of erroring
  (Android 11+) → guidance appended to the scan error.

## Native libs / C++ linkage

The Tauri CLI symlinks `libc++_shared.so` (from the NDK sysroot) and the built Rust
lib into `gen/android/app/src/main/jniLibs/<abi>/` automatically on every Android
build — no manual step, survives fresh clones. `src-tauri/build.rs` adds the
matching link-search + `-lc++_shared`.

## Signing

```
apksigner sign --ks release\resonance-compass.keystore --out release\<name>.apk ^
  src-tauri\gen\android\app\build\outputs\apk\universal\release\app-universal-release-unsigned.apk
adb install -r release\<name>.apk
```

## Known limitation / roadmap

A true "pick any folder" experience on Android needs a small custom Tauri mobile
plugin (Kotlin: `ACTION_OPEN_DOCUMENT_TREE` + `takePersistableUriPermission` +
`DocumentsContract` child enumeration). That also gives grants that survive
restarts without any manifest permission. Tracked as future work; the fixed
Music/Download scan covers the common case until then.
